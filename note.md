## 基本数据类型
* 整形

  - 向上/下取整 ``import math ;  math.ceil() ; math.floor();``

    - 多行字符串 ``用''',"""``

    - arr[1:5:2] 从1号位开始 到5-1=4号位结束，每隔2位截取

    - "zhan"+"san" 用+进行拼接，但是只能是相同数据类型

  - *tip*: 不同类型之间不能``+`` 
  - 类型转换
    - str --> int
      - 只有纯数字的整数字符串才能转成 int类型
    - float --> int ```直接截取整数部分```
    - bool  --> int ``True:1 ; false: 0``
    - float --> bool
    - float --> str
    - bool --> str
    - bool --> float

* 列表 (数组)
  - list=["1","2"]
  - 列表切片 同字符串
  - 列表的遍历
    - >for i, j in enumerate(d): # 枚举 i为下标
  - _列表常用方法_
    - max(), min(), len(), del() ```内置函数```
    - 新增 
      - append(val)
      - insert(index, val)
      - extend(val) ：将可迭代对象[str range() list]中的元素取出然后追加到列表尾部
    - 删除 
      - remove(val): 删除第一个值为 val的元素
      - pop(index) : 删除指定下标的元素 
      - del list[index] :删除指定下标的元素 
      - clear : 清空
    - 其他
      - [].index(val) : 查找val第一次出现的下标
      - [].count(val) : 统计val在列表中出现的次数
      - [].reverse()  : 列表逆置
      - [].sort(reverse = False) : 从小到大排序，reverse=False默认，如果=True就是从大到小排列 
  ``tip: 列表的这些方法都只是进行浅层操作，不会进入到嵌套元素中``
       
* 元组: 初始化后不能修改，也没有数组的常用方法

## 格式化输出

### 占位符的精度控制```%m.n```
  > m 正： 左对齐 负： 右对齐
  1. 字符串```%m.ns```
     ```
      m: 字符串的最小宽度，如果长度不够就补空格，如果m小于字符串长度就失效
      n：最多输出n个字符，n>字符串长度则不生效，如果小于则直接截断
     ```
  2. 浮点是```%m.nf```
    ```
      m:控制整体的宽度，如果位数不够就用空格补齐，如果m小就失效
      n:尾数精度控制，默认为6，如果n>尾数，则用0补齐，如果n<尾数，进行四舍五入
    ```
  3. 整数是```%m.nd```
    ```
      m: 整数的最小宽度，如果长度不够就补空格，如果m小于字符串长度就失效
      n: 精度控制，最少用n位显示整数，位数不够用0来补，否则就失效
    ```

## 函数

### 参数
- 位置参数[默认]： 按形参的位置传递参数，严格按照参数顺序来确定
- 关键字参数: 根据形参的关键字来传递，无顺序之分
  - 二者混用：位置参数必须在关键字参数之前
- 限制传参
  - ``/`` 前只能用位置参数
  - ``*`` 后只能用关键字参数
  - ``/``必须在 ``*`` 之前
- 参数默认值```必须作为末尾参数，放在所有必写参数的后面``

- 可变参数
  - 可变位置参数
    - ``*args``: args是元组类型
  - 可变关键字参数
    - ``**kwargs``: kwargs是字典类型
    - kwargs默认 {'args':'val'}
  - 可变参数可以和其他参数混用

### 函数的返回值
- ``return value``

### 函数作用域

- 全局作用域
- 局部作用域
  - 局部作用域和局部变量会在函数调用是创建，函数执行结束时销毁

### 函数的嵌套调用
  1. 函数的定义不会执行函数体中的代码，只有在函数执行时才会执行
    所以在testFuncNested()函数执行时才会去看内部的代码，此时所有的函数都以及定义了，所以可以正常执行
  2. 结果
     ```
     ------进入test1------
     ------进入test2------
     ------进入test3------
     ------正在执行test3------
     ------退出test3------
     ------退出test2------
     ------退出test1------ 
     原理就是： 函数调用栈
     ```

- > 函数的递归调用
  - 递归调用首先要确定函数的出口，即递归的终止条件
  - 函数调用自身的同时必须缩小问题规模
  - 递归调用的底层原理是函数调用栈也叫递归工作栈
